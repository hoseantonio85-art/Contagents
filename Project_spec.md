# Project Specification

## Goals
- Provide clear understanding of risks
- Reduce cognitive load
- Guide user to action

## Non-goals
- Not a BI dashboard
- Not a static reporting tool

## UX Rules
- One main focus per screen
- Avoid multiple competing charts
- Lists must be contextual (filtered)
- AI must explain, not duplicate UI

## Visual Rules
- Clean, minimal, modern (Apple/Yandex style)
- Soft UI (no heavy borders)
- Emphasis on hierarchy

## Interaction Rules
- Filters cascade from top to bottom
- Heatmaps and summaries control lists
- Lists are always reactive
